1) Чтобы найти ip злоумышленника я использовал утилиту wireshark на интерфейсе tap0. Далее я отследил в логах wireshark первый DHCP Offer пакет и взял source address (ip злоумышленника)
10.0.0.1

2) sudo ipconfig set tap0 automatic-v6 – автоматическое получение IPv6
3) Через wireshark позднее я увидел еще один DHCP Offer пакет.
Так как мы посылали DHCP Discover запрос по широковещательному каналу, то логично, что настоящий сервер тоже увидел наш запрос на выдачу ip адреса, но мы приняли наш ip уже от злоумышленника. 

Собственно source address в этом "запоздалом" пакете и есть адрес настоящего DHCP сервера - 10.64.222.14) Как я писал ранее, мы получаем свой ip адрес от DHCP сервера с ip == 10.0.0.1, чтобы этого избежать нужно как-то руками ограничить получение ip адреса от конкретного сервера, немного погуглилв, я увидел, что можно в /etc/dhcp/dhclient.conf добавить свои настройки для разных интерфейсов, в частности есть поле reject, которое игнорирует offer пакеты на указанных серверах.

Я прописал в файле /etc/dhcp/dhclient.conf следующие строки:

interface "tap0" {  reject 10.0.0.1}

Затем подключился к VPN, поднял интерфейс (ip link set dev tap0 up), 
и запустил команду dhclient -I tap0
