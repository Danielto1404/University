1) Выдать IPv4-адрес:* я зашел в файлик /etc/dhcp/dhcpd.conf для конфигурирования моего DHCP сервера IPv4 адресов.Прописал базовый конфигуратор для IPv4default-lease-time 600;max-lease-time 7200;# NETWROKsubnet 10.0.64.0 netmask 255.255.255.0 {   range 10.0.64.2 10.0.64.255;   option routers 10.0.64.1;}* добавил адрес роутинга к интерфейсу tap0:ip addr add 10.0.64.1 dev tap0* потом запустил сервер командой:/usr/sbin/dhcpd -4 -d -cf /etc/dhcp/dhcpd4.conf tap0(Более подробно все описал во втором подпункте так как его сделал первым)2) Выдать IPv6-адрес:	* я зашел в файлик /etc/dhcp/dhcpd6.conf для конфигурирования моего DHCP сервера IPv6 адресов.Прописал базовый конфигуратор для IPv6:# время в секундах на которое клиент получит адресdefault-lease-time 600;# максимально время в секундах на которе клиент получит адрес в случае запроса на его продлениеmax-lease-time 7200;   # обозначает сеть, в области которой будет работать данная группа настроекsubnet6 fd91:3abf:626b:64::/64 {           # промежуток из которого выдаются адреса     range6 fd91:3abf:626b:64:: fd91:3abf:626b:64:ffff:ffff:ffff:ffff;}* далее нужно было добавить адрес сети к интерфейсу tap0:ip addr add fd91:3abf:626b:64:: dev tap0* теперь можно запустить сервер на нужно интерфейсе:/usr/sbin/dhcpd -6 -d -cf etc/dhcp/dhcpd6.conf tap0/usr/sbin/dhcpd      - утилита запускающая сервер-6                   - IPv6 server mode-d                   - send logs to standard descriptor -cf                  - path to alternate configuration file flagetc/dhcp/dhcpd6.conf - my config filetap0                 - interface
